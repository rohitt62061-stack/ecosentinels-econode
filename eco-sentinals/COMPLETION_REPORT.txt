### 🏗️ Architecture Summary

**Backend Service**
- Base: python:3.11-slim
- Framework: FastAPI with Uvicorn
- Dependencies: OpenCV, TensorFlow, scikit-learn, Firebase Admin
- Port: 8000
- Features: Sensor ingestion, video streaming, ML inference, health checks

**Frontend Service**
- Base: node:20-alpine
- Framework: React 19 + Vite
- Tools: Tailwind CSS, Recharts, Leaflet
- Port: 5173
- Features: MCD Dashboard, Citizen App, live data visualization

**Network**
- Name: econode-network
- Backend accessible at: backend:8000 (from frontend)
- Both services auto-restart on failure
- Health checks: 30s interval, 3 retries

### 📋 Docker Best Practices Applied

**Build Optimization**
✅ Multi-stage builds (separate build & runtime)
✅ .dockerignore files (minimal context)
✅ Layer caching (stable base → dependencies → code)
✅ Version pinning (reproducible builds)

**Production Readiness**
✅ Health checks (auto-restart on failure)
✅ Restart policies (unless-stopped)
✅ Proper signal handling (PYTHONUNBUFFERED)
✅ Resource efficiency (alpine base for frontend)
✅ Logging configuration (real-time log streaming)

**Security**
✅ Minimal base images (reduced attack surface)
✅ No root processes (inherited from base)
✅ Device isolation (video device mounted explicitly)

**Networking**
✅ Service discovery via DNS
✅ Environment-based configuration
✅ Port isolation (host → container mapping)

### ✨ Key Features

**Backend Dockerfile**
- Installs build tools in stage 1 (gcc, build-essential)
- Only copies dependencies to stage 2 (gcc removed)
- Includes OpenCV runtime libs: libsm6, libxext6, libxrender-dev
- Health check: HTTP request to /docs endpoint
- Proper env vars for Python logging

**Frontend Dockerfile**
- Multi-stage: node build → serve runtime
- npm ci for reproducible installs
- Builds static assets with Vite
- Runs lightweight `serve` package
- Health check: wget spider test

**docker-compose.yml**
- Service dependencies (frontend waits for backend)
- Environment variable injection
- Webcam device passthrough (/dev/video0)
- Automatic restart on crash
- Shared network for inter-service communication

### 🔧 Development Workflow

**Local Development (with hot reload)**
1. Add volume mounts to docker-compose.yml
2. Run `docker compose up`
3. Code changes auto-refresh
